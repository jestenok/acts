description: docs.py принимает year параметром — не хардкодить год в документах
type: project
---

`docs.py` исправлен: функции `create_documents`, `create_act`, `create_tz` принимают параметр `year: int = 2025`.
`main.py` передаёт `YEAR` при вызове `create_documents`.

Год `2025` в `datetime.date(..., year=2025).strftime("%m")` трогать не нужно — он влияет только на форматирование месяца, не на вывод.

**Why:** до исправления год был захардкожен в тексте документов и именах файлов, документы за 2026 создавались с датой 2025.
**How to apply:** при генерации нового года убедиться что `YEAR` в `main.py` выставлен правильно — всё остальное подтянется автоматически.
