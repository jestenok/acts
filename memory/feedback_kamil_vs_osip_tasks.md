При генерации актов Камиля (jestenok1@gmail.com) нельзя использовать задачи из коммитов Осипа (kim2337334@gmail.com) — даже если они из тех же репозиториев.

Задачи по бизнес-логике salary (скидки, расчёты бонусов, отчёты, привязки менеджеров) — это Осипова зона. Камиль занимается серверной инфраструктурой (Helm, Flux, GitOps, PostgreSQL, MinIO, GitLab CI/CD) и своими фичами в calculator/planner.

**Why:** при добавлении «придуманных» задач были взяты реальные коммиты Осипа из salary-репо и приписаны Камилю — это некорректно, у каждого свои акты.
**How to apply:** при необходимости придумать задачи для Камиля — использовать только серверную тематику (Helm, Flux, PostgreSQL, MinIO, GitLab) или фичи, которые явно его (калькулятор, план/факт). Бизнес-логику зарплатного сервиса не трогать.
